# EventPhotos

Self-hostable event photo galleries. An organizer creates an event, uploads photos, and shares a public gallery ‚Äî attendees browse and view it from a link or QR code, no account required. Photographers and event organizers run it to hand galleries to their clients without depending on a third-party SaaS.

A photography business ([peakcapture.io](https://peakcapture.io)) runs live events on it.

## What it does

- **Public galleries** ‚Äî shareable per-event galleries with a lightbox, served over immutable-cached, ETag'd image endpoints. Anonymous ‚Äî viewing never touches a session or creates DB rows.
- **QR sharing** ‚Äî each event gets a QR code so attendees can open the gallery from a printed card or screen.
- **Async photo ingest** ‚Äî uploads are validated, deduplicated, and processed off the request path; EXIF is read in the event's timezone and thumbnails/previews are generated by a background worker.
- **Multi-organizer** ‚Äî role-based access (admin / organizer), with per-event ownership enforced by authorization voters.
- **Per-organizer email** ‚Äî each organizer configures their own SMTP transport (encrypted at rest); the app notifies attendees when an event's photos go live via double-opt-in subscriptions.
- **Event export/import** ‚Äî a portable archive (settings, photos, subscribers) to move an event between instances or owners.
- **Optional Google SSO** ‚Äî feature-flagged; disabled by default with zero config.

## Tech stack

PHP 8.5 ¬∑ Symfony 8 ¬∑ Doctrine ORM 3 / DBAL 4 ¬∑ PostgreSQL 16 ¬∑ Symfony Messenger (Doctrine transport) ¬∑ Flysystem ¬∑ Asset Mapper + Stimulus + Turbo + Tailwind (no Webpack) ¬∑ Docker Compose (php-fpm, nginx, Postgres, Mailpit, worker) ¬∑ GrumPHP ¬∑ PHPStan (level 10) ¬∑ Rector ¬∑ PHPUnit.

## Notable engineering

The parts of this codebase I'd point a reviewer at:

- **Idempotent async ingest pipeline.** Uploads compute a SHA-256 content hash and dedupe on a `(event, hash)` unique constraint. A `Photo` is a state machine (`Pending ‚Üí Ready | Failed ‚Üí Pending`); the Messenger handler no-ops unless the photo is `Pending`, so retries ‚Äî whether from the transport's backoff or a manual retry ‚Äî are always safe. Originals are never web-served; only generated derivatives are.

- **SSRF-hardened per-organizer SMTP.** Organizer mail credentials are encrypted at rest with libsodium `crypto_secretbox`. Transports are built through a factory that resolves the host, requires *every* resolved IP to pass a positive-allowlist public-IP classifier (rejecting RFC1918, loopback, link-local, CGNAT, and decoding IPv4-mapped/6to4/Teredo IPv6), then connects to the validated literal IP while passing the original hostname as the TLS peer name. This closes the validate-then-reconnect (DNS-rebinding) hole ‚Äî a transport that once verified but now resolves to a private address is hard-failed at send time, not trusted from a stored flag.

- **Authorization by voters.** Every state-changing action goes through an `EventVoter` / `PhotoVoter` / `EventCollectionVoter` ‚Äî admin bypass, otherwise an ownership check ‚Äî instead of scattered role strings.

- **Postgres-backed sessions** with rolling 30-day cookies, kept out of the Doctrine schema-validation gate via a schema filter, and bypassed entirely on anonymous public routes so galleries stay stateless.

- **Enforced quality bar.** PHPStan level 10, PSR-12, no magic numbers, copy-paste detection, Rector, a Roave security check, and `doctrine:schema:validate` against a freshly-migrated database all gate every commit locally (GrumPHP) and in CI (GitHub Actions + Postgres 16).

## Running it locally

```bash
docker compose up -d                 # app on http://localhost:8080
bin/console doctrine:migrations:migrate
bin/console app:create-user you@example.com "Your Name" <password> ROLE_ORGANIZER
```

The `worker` service consumes the async photo queue automatically. Tail it with `docker compose logs -f worker`; restart after handler changes with `docker compose restart worker`.

### Storage layout

- Originals: `var/uploads/photos/originals/event-<id>/<photoId>.jpg` ‚Äî private, never web-served
- Thumbnails: `var/uploads/photos/thumbs/...` ‚Äî served via `/p/<id>/thumb.jpg`
- Previews: `var/uploads/photos/previews/...` ‚Äî served via `/p/<id>/preview.jpg`

### Inspecting failed photo jobs

```bash
bin/console messenger:failed:show
bin/console messenger:failed:retry <id>
```

## Google sign-in (optional)

Google SSO is a login method, not a signup path ‚Äî accounts still come from `app:create-user` or invite redemption. It's fully optional: with `GOOGLE_OAUTH_CLIENT_ID` empty, the Google buttons disappear and `/oauth/google/*` returns 404. Setup guides: [`docs/setup/google-sso-dev.md`](docs/setup/google-sso-dev.md) and [`docs/setup/google-sso-prod.md`](docs/setup/google-sso-prod.md).
